Navigatie


Navigatie data

Zoals eerder gemeld zal de navigatie van de site afhangen van een aantal invoer parameters, gedeeltelijk gevoed door de informatie verkregen uit de SiteConfigurator en gedeeltelijk uit de object cloud. Per site zal er 1 navigatie instantie bestaan. Deze legt het verband vast tussen navigatie, template en inhoud vast. de navigatie kan gezien worden als een "view" op de data. Het vormt een sitemap.

De navigatie component


De navigatie component kan aan de hand van een gegeven navigatie + de huidige plek in de navigatie een aantal vragen beantwoorden:
-waar bevind ik mijn op de site
-wat is mijn broodkuimelpad
-gezien de wens mooie urls in de site te hebben zal het noodzakelijk zijn dat de navigatie component ook url's kan genereren.

De template chooser gebaseerd op navigatie

De template engine zal bij een binnen komend request gebruik makend van de beschikbare informatie moeten kiezen welke informatie beschikbaar zal moet worden. In het geval van de VPRO zal deze dus moeten kiezen over welke programma site het gaat en binnen de programma site moeten bepalen welk onderdeel er getoond dient te worden

De template chooser fallback(s)

In het geval template chooser niet tot een besluit kan komen dan kunnen er een aantal stappen worden ondernomen
-Url afbreken tot men wel op een bestaande pagina komt
-Als er in de parameters object nummers zitten kan worden gezocht of deze via de template engine toonbaar kunnen worden gemaakt
beiden gevallen leiden tot een redirect.
